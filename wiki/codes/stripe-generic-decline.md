# Stripe generic_decline

## Summary
Stripe's `generic_decline` code indicates the card was declined for an unknown reason, or that Stripe Radar or Adaptive Acceptance blocked the payment for risk management.

## Details
This is a catch-all decline code used when:
- The card issuer provides no specific decline reason
- Stripe's risk management systems (Radar, Adaptive Acceptance) block the transaction
- The specific decline reason should be kept private for security purposes

### Merchant response:
- Customer should contact their card issuer for more information
- Do not retry automatically
- Standard messaging: "Your card was declined. Please contact your card issuer."

### Security function:
This code is also used to mask sensitive decline reasons (like fraud detection) to prevent revealing anti-fraud mechanisms to potential bad actors.

## Technical reference
- **Stripe code**: `generic_decline`
- **Retry recommendation**: No
- **Generated by**: Card issuer (unknown reason) or Stripe risk systems
- **Related systems**: Stripe Radar, Adaptive Acceptance
- **Customer action**: Contact card issuer
