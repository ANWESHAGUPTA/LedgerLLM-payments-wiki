# Stripe fraudulent

## Summary
Stripe's `fraudulent` decline code indicates the payment was declined because Stripe's fraud detection systems suspect fraudulent activity.

## Details
This decline is generated by Stripe's internal fraud detection mechanisms, not by the card issuer. It represents Stripe's assessment that the transaction poses a high fraud risk based on various signals and patterns.

### Merchant handling:
- **Critical**: Do not reveal the specific decline reason to the customer
- Present the decline in the same manner as `generic_decline`
- This prevents fraudsters from learning about detection methods
- Do not retry the transaction

### Security considerations:
- Revealing fraud detection triggers helps fraudsters refine their techniques
- Generic messaging protects merchant and payment ecosystem
- Transaction should be logged for fraud analysis

## Technical reference
- **Stripe code**: `fraudulent`
- **Retry recommendation**: No
- **Customer messaging**: Present as generic decline
- **Generated by**: Stripe fraud detection, not card issuer
- **Related systems**: Stripe Radar, Adaptive Acceptance
